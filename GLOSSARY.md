## A. MD シミュレーションの基礎

### 分子動力学 (Molecular Dynamics, MD)

「**全原子に Newton の運動方程式 F = ma を解いて、時間発展を追いかける**」コンピュータシミュレーション。

```
時刻 t の座標と速度
        │
        ▼ 力場から各原子の力 F を計算
        │
        ▼ a = F/m → 速度を更新 → 座標を更新
        │
時刻 t + Δt の座標と速度
```

これを 100 万回くらい繰り返して、原子がどう動くかを観測します。Δt は通常 1〜2 fs (= 10⁻¹⁵ 秒)。

### アラニンジペプチド (alanine dipeptide)

アミノ酸 1 個 (アラニン) の両端を **ACE** (アセチルキャップ) と **NME** (N-メチルキャップ) で塞いだ最小ペプチドモデル。

```
   ACE        ALA       NME
  ┌────┐    ┌────┐    ┌────┐
  CH3-CO----NH-CHα-CO----NH-CH3
              │
              CH3 (側鎖)
```

なぜチュートリアルで頻出か:

- **小さい** (原子数 22 個 + 水) ので計算が軽い
- **両端がキャップされている** → タンパク質中のアミノ酸残基と化学環境が似ている
- **二面角 φ/ψ が 1 セットだけ** あるので Ramachandran プロットの教科書例として最適

### キャップ (capping group)

ペプチドを切り出すと N 末端は `-NH3+`、C 末端は `-COO-` の電荷を持ちます。これだと「タンパク質の途中の残基」とは挙動が変わります。そこで小さな分子で塞いで電荷を中和したものがキャップ:

- **ACE** (アセチル) — N 末端側、`CH3-CO-`
- **NME** (N-メチル) — C 末端側、`-NH-CH3`

### 残基 (residue)

タンパク質を構成する 1 単位 (アミノ酸 1 つ分)。AMBER の入力ファイルでは `:1` で 1 番目の残基、`:2` で 2 番目、のように指定する。今回:

| 番号 | 残基名 |
|------|--------|
| `:1` | ACE |
| `:2` | ALA |
| `:3` | NME |

### 二面角 (dihedral angle)

4 原子 A-B-C-D が決める「ねじれの角度」。タンパク質の主鎖では:

- **φ (phi)** — C(i-1)-N(i)-Cα(i)-C(i)
- **ψ (psi)** — N(i)-Cα(i)-C(i)-N(i+1)

φ と ψ の組み合わせがタンパク質の二次構造 (α ヘリックス、β シート) を決めます。

### Ramachandran プロット

横軸 φ、縦軸 ψ で点を打った図。アラニンジペプチドの MD では「どの (φ, ψ) を訪問したか」のヒストグラムが取れる。

## B. 力場とエネルギー

### 力場 (force field)

各原子に働く力を計算するためのエネルギー関数 + その関数のパラメータ群。AMBER 系の典型的な式は:

```
E_total = Σ_bonds   K_b (r - r_eq)²            ← 結合の伸縮 (バネ)
        + Σ_angles  K_θ (θ - θ_eq)²            ← 結合角
        + Σ_dihed   Σ_n  V_n/2 [1 + cos(nφ - γ)]  ← 二面角
        + Σ_ij      ε_ij [(σ/r)¹² - 2(σ/r)⁶]   ← van der Waals (LJ)
        + Σ_ij      q_i q_j / (4πε₀ r_ij)       ← 静電 (クーロン)
```

「結合をバネで近似、ねじれを cos の和で近似、原子間反発と引力は Lennard-Jones、電気は素朴な点電荷」という割り切り。**量子効果は基本的に無視**するので、結合の切断・形成は扱えない (= 化学反応はできない)。

### ff19SB

タンパク質用の AMBER 力場 (2020)。実験 NMR データ (二次元 NMR で測れる J-coupling など) に backbone の挙動が合うようにフィッティングされている。**OPC 水と組み合わせて使うように調整されている**点が大事。

### OPC 水

4 点モデルの水 (実際の原子は H, H, O の 3 つだが、電荷を持つ「ダミー粒子」を 1 つ加えて電荷分布を改良)。TIP3P (3 点) より自己拡散係数や誘電率の再現性が高い。`leaprc.water.opc` で読み込む。

### 静電相互作用 (electrostatics)

クーロン則 `F = q₁q₂ / (4πε₀ r²)` で計算される長距離相互作用。距離の 1 乗で減衰するので **遠くても無視できない**。MD では PME (下記) で扱う。

### van der Waals 相互作用

電荷を持たない原子同士でも働く弱い引力 + 強い反発 (近距離)。Lennard-Jones ポテンシャル `V(r) = 4ε[(σ/r)¹² - (σ/r)⁶]` で近似される。距離の 6 乗で減衰するので **8〜10 Å でカットオフしても OK**。

### Particle Mesh Ewald (PME)

周期境界条件下で長距離の静電相互作用を **O(N log N)** で正しく計算する手法。直接和は遠くまで足すと発散するので、

1. 直接空間 (近距離) は cutoff 以内をそのまま計算
2. 逆空間 (遠距離) は格子上で FFT で計算

の 2 段に分ける。AMBER ではデフォルトで ON。

### カットオフ (cutoff)

「これより遠い原子間相互作用は (直接空間では) 計算しない」距離。AMBER のデフォルトは 8 Å 程度。**短いと精度低下、長いと計算コスト増**。

## C. 統計力学・アンサンブル

### アンサンブル (ensemble)

統計力学的に「何を一定に保つか」で分類される集団。MD でよく使うのは:

| 略号 | 意味 | 一定に保つ | 用途 |
|------|------|----------|------|
| **NVE** | micro-canonical | 粒子数 N、体積 V、エネルギー E | 純粋な数値検証用、ほぼ使わない |
| **NVT** | canonical | N、V、温度 T | 加熱、表面ありの系 |
| **NPT** | isothermal-isobaric | N、圧力 P、T | **本番 MD の定番** (実験条件 = 1 atm に近い) |

### 温度 (temperature) — MD における定義

運動エネルギーから定義される:

```
(3/2) N k_B T = (1/2) Σ m_i v_i²
```

つまり「全原子の運動エネルギーの平均」がそのまま温度に対応する (k_B = ボルツマン定数)。

### サーモスタット (thermostat)

系の温度を目標値に近づけるアルゴリズム。代表的なものに:

- **Berendsen** — 速度を一律スケール。簡単だが「正しい統計集団」にはならない (誤差が小さいが厳密ではない)
- **Langevin** — 摩擦 + ランダム力。**今回使う** (`ntt=3`)
- **Nosé-Hoover** — 仮想自由度を 1 つ追加して厳密に NVT を再現。実装やや複雑

### バロスタット (barostat)

圧力を目標値に近づけるアルゴリズム。

- **Berendsen** (`barostat=1`) — 体積を一律スケール。簡単で安定だが正しい揺らぎを与えない
- **Monte Carlo** (`barostat=2`) — 体積変化を MC で受理/棄却。AMBER の現代的な推奨

### 周期境界条件 (Periodic Boundary Condition, PBC)

系を「無限に並んだイメージ」として扱う境界条件。シミュレーション箱を出た原子は反対側から入ってくる:

```
┌───────────┐ ┌───────────┐ ┌───────────┐
│           │ │     ●→    │ │           │
│           │ │           │ │           │
└───────────┘ └───────────┘ └───────────┘
                    ▼
              ●→ → →
              そのまま隣の箱に入る = 元の箱の左端から入ってくる
```

これで「箱の表面で水が蒸発してしまう」みたいな表面効果を消せます。

### 切頭八面体 (truncated octahedron)

立方体の 8 隅を斜めに削ったような形の周期境界箱。同じ buffer 距離なら立方体より体積が小さい → **少ない水で済む = 速い**。AMBER の `solvateoct` で作れる。

## D. アルゴリズム

### エネルギー最小化 (energy minimization)

ポテンシャルエネルギーが小さくなる方向に原子をちょっとずつ動かす操作。MD と違って「時間発展」ではなく「最適化問題」。

#### Steepest Descent (最急降下法)

「今いる場所で一番急な下り」方向に進む。**遅いが、初期構造が滅茶苦茶でも壊れない**。

```
   E
   │ ●     初期構造 (高エネルギー)
   │
   │   ↓ 最急降下
   │      ●
   │           ●
   │_______________●_____ q (構造)
```

#### Conjugate Gradient (共役勾配法)

直前の方向を考慮して賢く下る。**速いが、初期エネルギーが高すぎると不安定**。

普通は steepest descent → conjugate gradient の順で使う (`maxcyc` と `ncyc` の使い分け)。

### Verlet 積分・Leap-frog

Newton 方程式を時間離散化して数値的に解く方法。AMBER ではリープフロッグ Verlet が使われている (時刻 t に座標、時刻 t+Δt/2 に速度を持つ)。

### SHAKE

X-H 結合 (例: O-H, C-H, N-H) は振動が速いので、本来は Δt = 0.5 fs くらいでないと発散する。SHAKE は **これらの結合長を解析的に固定** することで Δt = 2 fs を可能にするアルゴリズム。
- `ntc=2` で SHAKE オン
- `ntf=2` で「拘束された結合の力計算をスキップ」(ペアで使う)

### Langevin dynamics

通常の Newton 方程式に「摩擦項 -γv」と「ランダム力 R(t)」を足したもの:

```
m a = F(q) - m γ v + R(t)
```

これを使うと自動的に温度が `temp0` に保たれる (揺動散逸定理)。
- `ntt=3` で Langevin
- `gamma_ln` が摩擦係数 γ (単位 ps⁻¹)。**2.0 程度が標準**

## E. AMBER 固有の用語

### tleap

AMBER の「分子ビルダー」。力場ライブラリと水モデルを読み込んで、配列指定から系を組み立て、トポロジー (.parm7) と座標 (.rst7) を吐き出す。

### sander

AMBER の **MD エンジン (基本版)**。AmberTools にフリーで含まれている。

### pmemd

AMBER の **MD エンジン (高速版)**。CPU シリアル版は AmberTools に含まれる。MPI 並列の `pmemd.MPI` や GPU 版 `pmemd.cuda` は商用 AMBER のみ。
- `sander` と `pmemd` は同じ入力ファイルを読む。
- 物理は同じ。違いは速度と並列化。

### cpptraj

AmberTools の解析エンジン。RMSD、RMSF、水素結合数、二面角、クラスタリングなど何でもできる。スクリプト言語のインタプリタ的に使う。

### `&cntrl` ブロック

sander/pmemd の入力ファイルに書く「制御パラメータ群」。Fortran namelist 形式 (`変数名 = 値`、カンマ区切り、`/` で終わる)。

### `&wt` ブロック

`nmropt=1` のとき有効になる「特殊な制御」。本来は NMR 由来の距離拘束を入れるためのものだが、**チュートリアル 0 では「温度を時間経過で変える」用途に流用**している。

### `imin`, `ntx`, `irest`

実行モード関連:

| 変数 | 値 | 意味 |
|------|----|------|
| `imin` | 1 | エネルギー最小化 |
| `imin` | 0 | MD |
| `ntx` | 1 | 座標のみを読む (速度は乱数で初期化) |
| `ntx` | 5 | 座標 + 速度を読む (リスタート用) |
| `irest` | 0 | 新規実行 (時間カウンタを 0 から) |
| `irest` | 1 | リスタート (時間カウンタを引き継ぎ) |

### `ntb`, `ntp`

境界・圧力関連:

| 変数 | 値 | 意味 |
|------|----|------|
| `ntb` | 0 | 周期境界なし |
| `ntb` | 1 | 周期境界、体積一定 (NVT) |
| `ntb` | 2 | 周期境界、圧力一定 (NPT) |
| `ntp` | 0 | 圧力スケーリングなし |
| `ntp` | 1 | 等方スケーリング (箱の縦横を同時に伸縮) |

### `ntpr`, `ntwx`

出力頻度:

- `ntpr` — 何ステップごとにエネルギーをテキスト (`*.out`) に書くか
- `ntwx` — 何ステップごとにトラジェクトリ (`*.nc`) に座標を書くか

## F. ファイル形式

### `.parm7` (= `.prmtop`)

トポロジー + 力場パラメータファイル。中身: 各原子の質量・電荷・LJ パラメータ、結合・角度・二面角リスト、力場定数など。テキスト形式 (人間でも読める)。**MD 中ずっと使う**。

### `.rst7` (= `.inpcrd`)

座標ファイル (場合により速度・箱の大きさも含む)。MD のスタートと終わりに使う。バイナリ NetCDF と ASCII の 2 種類があり、AMBER は両対応。

### `.ncrst`

NetCDF 形式のリスタートファイル。座標 + 速度 + 箱寸法を全部含む。`-r` オプションで sander/pmemd が出す。次のステップの入力 (`-c`) に使う。

### `.nc`

NetCDF 形式の **トラジェクトリ** (時系列の座標列)。`-x` で出力。VMD/cpptraj で読める。

### `.out` (= `.mdout`)

sander/pmemd の人間向けログ。エネルギー、温度、圧力、密度などが定期的に書かれる。

### `.mdinfo`

MD 実行中の進捗情報。現在のステップ番号、推定残り時間など。ファイルは `-inf` で指定。`cat` で覗ける。

### `summary.*`

`process_mdout.perl` が `.out` から抽出して作る、各物理量の時系列ファイル (`summary.TEMP`, `summary.DENSITY`, `summary.ETOT` など)。`gnuplot` や `xmgrace` でそのまま読める 2 列テキスト形式。

## G. 解析・可視化

### RMSD (Root-Mean-Square Deviation)

参照構造との「位置ズレ」の平均的な大きさ:

```
        ┌───────────────────────────┐
        │  1                        │
RMSD = √│ ───  Σ_i  m_i (r_i - r_i^ref)²
        │  M                        │
        └───────────────────────────┘
```

- M は重みの総和 (mass-weighted のとき = 全原子の質量和、mass なしのとき = 原子数)
- 単位は Å

### `mass`-weighted RMSD

質量で重みを付けた RMSD。**重い原子のズレを重視する**。タンパク質の主鎖は C, N, O, S 等が中心なので、軽い水素のノイズに振り回されにくい。

### autoimage

cpptraj の機能。周期境界をまたいで「ワープしたように見える」分子を、シミュレーション箱の中に巻き戻す。**RMSD 計算前には必須** (やり忘れると周期境界をまたいだ瞬間に巨大な見せかけ RMSD が出る)。

### gnuplot

軽量な 2D プロットツール。本リポジトリで使う **xmgrace の代替**。

```bash
# ウィンドウで開く
gnuplot -p -e 'plot "summary.TEMP" with lines'

# PNG にエクスポート
gnuplot -e 'set terminal png; set output "out.png"; plot "summary.TEMP" with lines'

# 対話モード
gnuplot
gnuplot> set xlabel "time"
gnuplot> plot "summary.TEMP" with lines
gnuplot> exit
```

論文に出せる品質。多くの解析ソフトが gnuplot 用スクリプトを吐けるので汎用性が高い。

### xmgrace (Grace) — 公式チュートリアル原文で使われるが…

AMBER 公式チュートリアル原文では `xmgrace` を使っています。これは:

- 古い X11/Motif アプリケーション
- Apple Silicon Mac の conda-forge には**ビルドされていない**
- Linux ではまだ広く使われている

本リポジトリでは可搬性のため `gnuplot` で代用しています。**機能は同等**で、`summary.TEMP` のような 2 列テキストファイルをグラフ化する用途に違いはありません。Linux 環境を使っていて xmgrace が入る場合は、`xmgrace summary.TEMP` でも同じグラフが見られます。

### VMD (Visual Molecular Dynamics)

3D 分子ビューア。タンパク質と水を分けて描画したり、トラジェクトリを再生したりできる。**Nix では扱いにくいので別途インストール**を推奨。

## H. 単位の早見表

| 物理量 | 単位 | 備考 |
|--------|------|------|
| 距離 | Å (オングストローム) | 1 Å = 10⁻¹⁰ m。原子のサイズが約 1 Å |
| 時間 | ps (ピコ秒) | 1 ps = 10⁻¹² s。fs (フェムト秒、10⁻¹⁵ s) もよく出る |
| エネルギー | kcal/mol | 1 kcal/mol ≈ 4.184 kJ/mol。室温 RT ≈ 0.6 kcal/mol |
| 温度 | K (ケルビン) | 室温 = 298〜300 K |
| 圧力 | atm or bar | 1 atm ≈ 1 bar。MD では bar が出ることも |
| 質量 | amu (= Da) | 水素 = 1, 炭素 = 12 |
| 力 | kcal/(mol·Å) | F = -dE/dr の自然単位 |
| 摩擦 (γ) | ps⁻¹ | Langevin の `gamma_ln` |

### 役立つ目安

| シーン | 目安 |
|--------|------|
| MD 時間刻み | 2 fs (= 0.002 ps) |
| 加熱 | 0 → 300 K を 20 ps |
| 平衡化 | 0.1〜1 ns |
| 本番 MD (タンパク質) | 100 ns 〜 数 μs |
| 1 ns に必要な計算ステップ | 500,000 ステップ (dt=2fs) |
| アラニンジペプチドの 10 ns (CPU) | 公式 ≒ 13 時間 |
