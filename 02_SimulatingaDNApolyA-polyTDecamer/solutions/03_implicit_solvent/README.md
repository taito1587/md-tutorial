# Step 03 — Implicit Solvent MD (GB 暗黙溶媒)

> 必要な前知識: Step 01, 02 が完了していること。詳細は [GLOSSARY](../../../GLOSSARY.md)

## このステップで何をやるか

DNA を **GB (Generalized Born)** という溶媒モデルで MD します。「**水分子を陽 (あらわ) に置かずに、溶媒効果を平均場として近似的に含める**」手法。**真空 MD と明示溶媒 MD の中間的な選択肢**です。

具体的なアウトプット:

- `min.ncrst` — 最小化後の座標
- `md.nc`, `md.out` — GB MD のトラジェクトリとログ

トポロジーは Step 01 で作った **`polyAT_vac.prmtop` を再利用** します (GB は溶媒原子を必要としないので、真空用と同じトポロジーで動かせる)。

## 1. 暗黙溶媒 vs 明示溶媒 — まずは直感

### 明示溶媒 (explicit solvent)

```
   ●●●●●●●●●●●●●●●●●●●●●●●●     ← 水分子をひとつひとつ MD で動かす
   ● ●  ● ●  ● ●  ● ● ● ●●
   ●  ●  ┃DNA┃  ●  ●  ●  ●     例: 3,000 個の水分子 (= 9,000 原子)
   ●  ●  ┃   ┃  ●  ●  ●  ●
   ●●●●●●●●●●●●●●●●●●●●●●●●
```

水分子の動きを 1 個ずつ追跡 → **正確だが計算が重い**。

### 暗黙溶媒 (implicit solvent / GB)

```
                                ← 水分子は描かれていない (= 計算しない)
        ┃DNA┃                    代わりに、DNA 周辺に「水があったら
        ┃   ┃ + 数式で           こうなるはず」の効果を**式で**追加
        補正                     例: 水分子 0 個 (← 計算が大幅に軽い)
```

水分子の効果を **方程式に組み込む** → **速いが粗い**。

### どっちが優れている?

「目的による」が答え。

| | 明示溶媒 | 暗黙溶媒 (GB) |
|---|---|---|
| 速度 | 遅い (×1) | **速い (×10〜50)** |
| 精度 | ★高い | 中 |
| 水のミクロ構造 (1 分子の動き) | 見える | 見えない |
| マクロな溶媒効果 (誘電応答) | 見える | **見える** |
| 大きな構造変化 (フォールディング等) のドライビング | 正確 | やや歪む |
| 用途 | 最終解析、ベンチマーク | 大量サンプリング、スクリーニング |

「**どっちでも溶媒効果は入る、でもアプローチが全然違う**」と覚えるのが大事。

## 2. GB (Generalized Born) のもう少し詳しい話

### Born の元の式 (1920 年)

電荷 q を持つ半径 R の球 (= 原子 1 個) を、誘電率 ε の溶媒に入れたときのエネルギー安定化:

```
ΔG_solv = -(q² / 8π ε₀) × (1 - 1/ε) × (1/R)
```

物理的な意味:
- **電荷が大きい** (q²) → 溶媒和エネルギー大
- **半径が小さい** (1/R) → 溶媒和エネルギー大 (電場が強いから)
- **溶媒の誘電率が大きい** (ε) → 溶媒和エネルギー大 (応答が強い)

水の ε ≈ 78.5 なので、`(1 - 1/78.5) ≈ 0.987`。約 99% の溶媒和効果が得られる。

### 多原子分子への拡張 (= GB)

普通の分子は 1 原子ではなく多原子 (DNA なら数百原子)。**Generalized Born** はこれを近似:

```
ΔG_solv = -(1/8π ε₀)(1 - 1/ε) × Σ_{ij} qᵢqⱼ / f(rᵢⱼ, αᵢ, αⱼ)
```

- `qᵢ` は原子 i の電荷
- `αᵢ` は原子 i の **有効 Born 半径** (周りの原子による遮蔽を考慮した実効半径)
- `f(...)` は距離 + 有効半径から決まる関数

ポイントは「**原子ペア (i, j) について和を取る**」こと。**水の効果が、原子間の項として入ってくる**。これにより水分子を陽に置かなくても水の影響が出る、というカラクリ。

## 3. AMBER の `igb` パラメータ

AMBER は何種類かの GB 実装を提供しています。`igb` の数字で選ぶ:

| `igb` | モデル名 | 公開年 | 特徴 |
|---|---|---|---|
| 0 | (なし) | — | GB 無効 (= 真空または明示溶媒) |
| **1** | **HCT** (Hawkins-Cramer-Truhlar) | 1996 | 最古典・最速。**今回これを使う** |
| 2 | OBC1 (Onufriev-Bashford-Case) | 2004 | やや改良、タンパク質で標準的 |
| 5 | OBC2 | 2004 | OBC1 の別パラメータセット |
| 7 | GBn | 2007 | より新しい近似 |
| 8 | GBn2 | 2012 | 最新世代 |

「数字大きい = 新しい = 精度高い」が、計算速度は若干落ちる。**今回はチュートリアルに合わせて `igb=1`** を使います。実用研究では `igb=5` か `igb=8` が使われがち。

## 4. 入力ファイルの解説

### `min.in` (最小化)

```
 &cntrl
  imin   = 1, maxcyc = 500, ncyc = 250,
  ntb    = 0,        ! 周期境界なし (GB は周期不要)
  igb    = 1,        ! HCT GB モデル
  cut    = 12        ! cutoff
 /
```

| パラメータ | 値 | 意味 |
|---|---|---|
| `imin` | `1` | 最小化 |
| `maxcyc/ncyc` | `500/250` | 章 01 と同じパターン |
| `ntb` | `0` | **周期境界なし** (GB なので) |
| `igb` | `1` | HCT GB モデル ← ★ここが新しい |
| `cut` | `12` | cutoff (章 01 は 8 Å、章 02 は教育目的で大きめ) |

### `md.in` (MD 本体)

```
 &cntrl
  imin     = 0,
  ntb      = 0,
  igb      = 1,
  ntpr     = 100, ntwx = 100,
  ntt      = 3, gamma_ln = 1.0,
  tempi    = 300.0, temp0 = 300.0,
  nstlim   = 100000, dt = 0.001,
  cut      = 12.0
 /
```

| 項目 | 値 | 意味 |
|---|---|---|
| `dt` | `0.001` | **1 fs** (SHAKE 不使用なので 2 fs は危険) |
| `tempi`, `temp0` | `300.0` | **加熱フェーズなし** で直接 300 K 開始 |
| `nstlim × dt` | 100,000 × 0.001 | 100 ps 全体 |
| `ntt`, `gamma_ln` | `3`, `1.0` | Langevin (章 01 は 2.0、ここは 1.0) |

### 真空 MD (Step 02) との違い

たった 1 つ:

| | Step 02 (真空) | Step 03 (GB) |
|---|---|---|
| `igb` | **0** (溶媒なし) | **1** (GB あり) |
| 他 | 同じ | 同じ |

これだけで全然違う挙動になるのが面白いところ。

## 5. 実行してみる

```bash
cd workspace
mkdir -p 03_implicit_solvent && cd 03_implicit_solvent

cp ../../solutions/03_implicit_solvent/min.in .
cp ../../solutions/03_implicit_solvent/md.in .

PARM=../01_setup/polyAT_vac.prmtop   # ★GB は水なしの prmtop で OK
INIT=../01_setup/polyAT_vac.rst7

# (a) GB 最小化
sander -O -i min.in -o min.out -p $PARM -c $INIT -r min.ncrst

# (b) GB MD
sander -O -i md.in -o md.out -p $PARM -c min.ncrst -r md.ncrst -x md.nc
```

100 ps の GB MD は CPU で **10〜30 分** くらい (真空より少し遅い、明示溶媒よりはるかに速い)。

## 6. 出力の見方

```
03_implicit_solvent/
├── min.in / min.out / min.ncrst
└── md.in / md.out / md.ncrst / md.nc
```

### `md.out` で確認するポイント

```bash
# 温度の推移
grep "TEMP(K)" md.out | awk '{print $4, $9}' | head -20
```

期待: **300 ± 5 K** で安定して振動。

```bash
# エネルギーの GB 寄与
grep "EGB" md.out | head -10
```

`EGB` は GB 由来のエネルギー (Born ソルベーション)。**負の値**で、**水分子による DNA の安定化を表現**しています。

### 期待される観察

- **真空 MD では崩壊した DNA が、GB ではずっと安定** (RMSD 2-3 Å 程度)
- これは「Born ソルベーションの安定化」が効いている証拠
- ただし水分子由来の細かい揺らぎは見えない (静的な感じになる)

具体的な比較は Step 05 で 4 つのトラジェクトリ (真空 ×2 + GB + 明示溶媒) を重ねたグラフで一目瞭然になります。

## 7. なぜ GB は便利なのか — もう少し実用的な話

### 計算速度の比較

10 bp DNA の 1 ns MD を 1 CPU コアで回した場合:

| モデル | 原子数 | 所要時間 |
|---|---|---|
| 真空 | ~640 (DNA だけ) | 数分 |
| **GB** | **~640 (DNA だけ)** | **十数分** |
| 明示溶媒 (TIP3P + 8 Å buffer) | ~10,000 (DNA + 数千の水) | 数時間 |

**1 桁速い** ことがわかります。

### こんなときに GB が便利

- **大量に MD を回したい** (異なる初期構造を 100 個比較、薬剤候補のスクリーニング、etc.)
- **長時間 MD でフォールディング過程を見たい** (μs オーダーまで届く)
- **マシンリソースが限られている** (ノート PC で論文を書きたい、研究室の共有 GPU が混んでる)

### こんなときは GB を避ける

- **水分子の局所構造が重要** (例: タンパク質-リガンド結合界面の水分子 1 個 1 個の役割)
- **DNA-イオンの相互作用が重要** (溶媒の誘電応答だけではイオンの動きが正しく出ない)
- **論文で "explicit solvent" がデファクトの分野** (構造生物学の最先端は通常明示溶媒)

## 9. 用語ミニ辞典

- **GB (Generalized Born)**: 多原子分子の溶媒和エネルギーを近似する手法。詳細 [GLOSSARY](../../../GLOSSARY.md)
- **Born 半径 / 有効 Born 半径**: 原子の電荷が感じる「実効的な誘電半径」。周りの原子に遮蔽されると小さくなる
- **暗黙溶媒 (implicit solvent)**: 水分子を陽に置かず、平均場として効果を入れる手法の総称 (GB, PB など)
- **明示溶媒 (explicit solvent)**: 水分子を 1 個ずつ MD で動かす方式
- **誘電率 (dielectric constant)**: 電場に対する物質の応答の強さ。水は ε=78.5
