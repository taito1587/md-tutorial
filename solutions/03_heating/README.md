# Step 03 — Heating (0 K → 300 K にゆっくり加熱)

> このステップで必要な前知識: [PRIMER](../../PRIMER.md), Step 02 が完了していること

## このステップで何をやるか

最小化が終わった「**動いていない構造**」に、**温度 0 K から 300 K まで徐々に上げながら 20 ps の MD** を走らせます。これで初めて系が「動き出します」。

具体的なアウトプット:

- `02_Heat.ncrst` — 加熱後の座標 + 速度 (= 300 K で平衡化された状態)
- `02_Heat.nc` — トラジェクトリ (時系列の座標、20 ps 分)
- `02_Heat.out` — エネルギー・温度のログ

---

## 1. 前提知識 — そもそも MD における「温度」って?

### 統計力学的な定義

古典統計力学では、温度 T は **全原子の運動エネルギーの平均** で定義されます:

```
(3/2) N k_B T = (1/2) Σ_i m_i v_i²
                  └────────────────┘
                  全原子の運動エネルギー
```

ここで:
- `N` = 自由度の数 (≒ 全原子数 × 3、SHAKE で減ることもある)
- `k_B` = ボルツマン定数 (1.987 × 10⁻³ kcal/mol·K)
- `m_i, v_i` = 原子 i の質量と速度

つまり「**速く動いている原子が多い = 温度が高い**」というシンプルな関係。

### MD での「加熱」とは

最小化終了時点では速度がすべて 0 です (= 0 K)。これに **少しずつランダムな速度を分配** していくと、温度が 0 → 数十 → 数百 K と上がっていきます。これが「加熱 (heating)」。

---

## 2. なぜ「ゆっくり」加熱するのか

### いきなり 300 K で MD を始めると

「速度を分配すると言ったって、初期構造に直接 300 K 相当の速度を与えればいいじゃん」と思うかもしれません。実際そうしてもいいのですが:

```
構造のひずんだ部分 (= 力が大きいところ)
    │
    ▼
速度を入れた直後、その部分の原子が大きく加速
    │
    ▼
他の原子と激突、エネルギー局所集中 (ホットスポット)
    │
    ▼
シミュレーション発散 (vlimit exceeded)
```

ということが現実によく起こります。これを避けるため、**温度を時間と共に徐々に上げる** のがセオリー。

### 加熱プロファイル

このチュートリアルでは:

```
Temperature
    │
300 K┤              ┌──────────────  ← 最後 1 ps は 300 K で平衡化
    │             ╱
    │           ╱
    │         ╱   ← 18 ps かけて線形ランプ
    │       ╱
    │     ╱
    │   ╱
  0 K┤ ╱
    └────────────────────────── time
    0      9 ps        10 ps     20 ps
                       (= 9000 step) (= 10000 step)
```

`&wt` ブロックで「ステップ 0 から 9000 まで線形に 0 K → 300 K にせよ」と指示しています。

---

## 3. NVT アンサンブルを選ぶ理由

### なぜ NPT じゃなくて NVT か

加熱中は**密度が大きく変動**します (温度が上がると体積が膨張する)。NPT (圧力一定) で加熱すると、箱がガクガク変形して数値が不安定になります。なので

- **加熱中は NVT** (体積一定) で温度だけ気にする
- **本番 MD は NPT** (圧力一定 = 1 atm) で実験条件に合わせる

の使い分けが定石。

## 4. Langevin サーモスタット — 温度を制御する仕組み

### 何をするものか

「全原子の運動エネルギー」が目標温度 `temp0` に対応する値になるように、速度を補正するアルゴリズムが**サーモスタット**です。Langevin はその一種で、**仮想的な「摩擦」と「ランダム力」を足す**形で実装されます。

通常の Newton 方程式:

```
m a = F
```

Langevin 方程式:

```
m a = F - m γ v + R(t)
       ↑       ↑
       摩擦     ランダム力
```

ここで:
- `γ` (gamma_ln) は摩擦係数 [ps⁻¹]
- `R(t)` は揺動散逸定理から自動的に温度 `temp0` を実現するように振幅が決まるランダム力

### 物理的なイメージ

「**透明な水 (heat bath) の中で粒子が泳いでいる**」と思うとよいです。

- 摩擦 → 速すぎる粒子は減速される (→ 温度が上がりすぎない)
- ランダム力 → 遅すぎる粒子は熱浴から押されて加速される (→ 温度が下がりすぎない)

この 2 つのバランスで、自動的に温度が `temp0` に収束します。

### `gamma_ln=2.0` の選び方

| `gamma_ln` (ps⁻¹) | 挙動 |
|------|------|
| 0.5〜1.0 | 弱い (温度制御は緩いが、動力学はほぼ Newtonian) |
| **2.0〜5.0** | **標準** |
| 10 以上 | 強い (温度はビシッと制御されるが、動力学が遅くなる) |

「**動力学を強く乱したくない**」かつ「**温度はちゃんと制御したい**」のバランスで 2.0 が一般的。

### 他のサーモスタット

| `ntt` | 名称 | 特徴 |
|-------|------|------|
| 0 | NVE (なし) | 温度制御なし、エネルギー保存 |
| 1 | Berendsen | 速度を一律スケール。簡単だが「正しい統計集団」にならない |
| 2 | Andersen | ランダムに原子の速度を再アサイン |
| **3** | **Langevin** | **今回使う**。動力学を保ちつつ温度制御 |
| 9 | Langevin (改良版) | より高度な実装 |

## 5. SHAKE — 2 fs 刻みを可能にするトリック

### 問題: 水素を含む結合は振動が速い

C-H, O-H, N-H などの結合は周期 ≒ 10 fs で振動します。Nyquist の定理的に、この振動を捉えるには **dt < 0.5 fs** くらいでないと数値積分が発散します。

しかし dt = 0.5 fs だと、10 ns の MD に **20,000,000 ステップ** も必要になります (重い)。

### 解: 結合長を解析的に固定

「**水素を含む結合は伸び縮みしない、と決めてしまう**」のが SHAKE。

- 物理的に: 高周波数振動 (= 水素結合の伸縮) は系の重要な動きには寄与していない (= ノイズ的)
- 数値的に: その振動を計算しなくて済むので、**dt を 2 fs まで伸ばせる**
- アルゴリズム的に: 各ステップで結合長制約を満たすよう座標を補正する

`ntc=2` で SHAKE 有効化、`ntf=2` で「拘束した結合の力計算もスキップ」というペアで使います。

## 6. `&wt` ブロック — 温度ランプの仕組み

### 普通の使い方

`&wt` (= "weights") はもともと **NMR 由来の距離拘束** をかけるためのブロック:

```
&wt type='REST', value=10.0 /
&wt type='END' /
```

みたいに使います。

### このチュートリアルでの活用

ところがその仕組みを流用して **`temp0` を時間で変える** ことができます:

```
&wt type='TEMP0', istep1=0,    istep2=9000,  value1=0.0,   value2=300.0 /
&wt type='TEMP0', istep1=9001, istep2=10000, value1=300.0, value2=300.0 /
&wt type='END' /
```

意味:
- 1 行目: ステップ 0 から 9000 までの間、`temp0` を 0.0 → 300.0 に **線形補間** で動かす
- 2 行目: ステップ 9001 から 10000 までは `temp0` を 300.0 で固定 (= 平衡化)
- 3 行目: ブロック終了マーカー

これにより「**18 ps かけて 0 → 300 K**」「**最後の 2 ps は 300 K で平衡化**」が実現されます。

### `nmropt=1` を忘れずに

`&wt` ブロックは `nmropt=1` のときだけ有効になります。`&cntrl` ブロックの中に必ず書いてください。

### `02_Heat.out` のタイムステップ表示

100 ステップごとに以下のブロックが出ます:

```
 NSTEP =      100   TIME(PS) =    0.200    TEMP(K) =   34.21    PRESS = ...
 Etot   =     ...   EKtot   =    34.5     EPtot  =   ...
 BOND   =     ...   ANGLE   =   ...        DIHED  =   ...
 1-4 NB =     ...   1-4 EEL =   ...        VDWAALS=   ...
 EELEC  =     ...   EHBOND  =   0.0        RESTRAINT=   0.0
 EAMBER (non-restraint) =   ...
```

各項目:

| 項目 | 意味 |
|------|------|
| `NSTEP` | 何ステップ目か |
| `TIME(PS)` | 開始からの時間 (ps) |
| `TEMP(K)` | **このステップの瞬間温度**。ノイズで揺れるが、徐々に 300 K に近づく |
| `PRESS` | 瞬間圧力 (NVT なので意味は薄い) |
| `Etot` | 全エネルギー (= EKtot + EPtot) |
| `EKtot` | 全運動エネルギー (温度に対応) |
| `EPtot` | 全ポテンシャルエネルギー |
| 残り | エネルギー成分の内訳 (Step 02 と同じ) |

### 健全な加熱パターン

```bash
# 温度の推移だけ抜き出す
grep "TIME(PS)" 02_Heat.out | awk '{print $4, $9}'
```

期待される出力 (例):
```
0.200   34.21
0.400   58.92
0.600   84.65
...
9.000   299.85   ← 9 ps 時点で約 300 K
9.200   300.10
...
20.000  299.45
```

ポイント:

- ✅ 0 から 300 まで **滑らかに上昇**
- ✅ 9 ps を過ぎると **300 ± 数 K で振動**
- ✅ 大きなスパイクや急激な変動が **ない**

## 10. つまずきポイント

| 症状 | 原因と対処 |
|------|-----------|
| `vlimit exceeded` で死ぬ | 加熱が急すぎる。`&wt` の `istep2` を伸ばす (例: 9000 → 18000)。または最小化が不十分 |
| 温度が 300 K に達しない | `gamma_ln` が大きすぎて系が固まっている。1.0 に下げる |
| `BOND` が異常に大きい | SHAKE が効いていない可能性。`ntc=2`, `ntf=2` を確認 |
| `PRESS` が異常 | 加熱中の圧力は不安定で normal、無視 OK |
