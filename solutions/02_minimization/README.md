# Step 02 — Energy minimization (エネルギー最小化)

> このステップで必要な前知識: [PRIMER](../../PRIMER.md), Step 01 が完了していること

## このステップで何をやるか

前ステップ 01 で `tleap` が組んだ初期構造には、原子同士がぶつかりかけている部分や、結合長が平衡値からズレている部分があります。そのまま MD を走らせると数式中に巨大な値が現れ、シミュレーションが**爆発します** (`vlimit exceeded` エラー)。

そこで MD の前に「エネルギーが小さくなる方向に少しずつ原子を動かして、ひずみを抜く」 — これが**エネルギー最小化** (energy minimization) です。

具体的なアウトプット:

- `01_Min.ncrst` — 最小化後の座標
- `01_Min.out` — 各サイクルのエネルギー履歴ログ

---

## 1. 前提知識 — そもそもエネルギーって何?

### MD の文脈での「エネルギー」

[力場](../../GLOSSARY.md#力場-force-field) の式で計算される **ポテンシャルエネルギー**:

```
E(r) = (結合の伸縮の歪み)
     + (結合角の歪み)
     + (二面角のねじれ)
     + (van der Waals 反発・引力)
     + (静電エネルギー)
```

ここで `r` は全原子の座標を縦に並べた巨大なベクトル (3 × N 次元、N = 原子数)。エネルギーは **「座標から決まる数値」** です。「**ポテンシャルエネルギー曲面 (potential energy surface, PES)**」とも言います。

### 最小化とは「PES を下る」こと

```
   E
   │\        ← 初期構造 (高エネルギー、ぶつかり気味)
   │ \
   │  \    最小化アルゴリズムで
   │   \____ ↓ 谷の底まで滑り下りる
   │       \___
   │           \___●  ← 最終構造 (谷底、安定)
   │_______________________ q (構造)
```

物理的には「**力 F = -dE/dr が 0 になる点を探す**」操作。MD ではないので時間は流れません。**速度・温度の概念がない**ことに注意。

---

## 2. なぜ最小化が必要なのか

### tleap の出力は「机上の構造」

`tleap` は溶質を箱の中央に置いて、その周りに水分子テンプレート (OPCBOX) をパタパタと並べただけです。なので例えば:

- 溶質側鎖の H と水の H が **0.8 Å しか離れていない** (本来 2.5 Å くらい欲しい) → 巨大な反発力
- 水分子同士が偶然近すぎる位置に来ている

ということが普通に起こります。これらを **MD 前に解消** しないと、初手で系がぶっ飛びます。

### 最小化 vs MD の違い

| | 最小化 | MD |
|---|---|---|
| 何を解く? | dE/dr = 0 (停留点) | F = ma の運動方程式 |
| 時間の概念 | なし (cycles で数える) | あり (fs 単位で進む) |
| 速度・温度 | なし | あり |
| 統計的な意味 | 1 つの局所安定構造 | アンサンブルからのサンプリング |

---

## 3. 最小化アルゴリズムの直観

このステップでは 2 種類のアルゴリズムを **順番に** 使います。

### Steepest Descent (最急降下法) — 前半 1000 cycle

「**今いる場所で一番急な下り**」方向に少しずつ進む。

```
       E
       │
       │  ●          ← 初期点
       │  ↓ -∇E (この向き)
       │  ●
       │  ↓
       │   ●
       │    ↓
       │      ●_____ ← 谷底に近づく
```

特徴:
- ✅ **どんなにグチャグチャな初期構造でも壊れない**
- ❌ 谷底近くではジグザグして進みにくい (収束が遅い)

### Conjugate Gradient (共役勾配法) — 後半 1000 cycle

直前の探索方向と「直交補正」した方向に進む賢い方法。

特徴:
- ✅ 谷底近くで **はるかに速く収束**
- ❌ 初期エネルギーが高すぎると **不安定**

### この組み合わせの定石

「**最初は steepest で安全にエネルギーを落とす → 後半は conjugate で精密に谷底へ**」が AMBER の伝統的な定石。`maxcyc=2000`, `ncyc=1000` の意味は:

- `maxcyc=2000`: 全部で 2000 cycle まで
- `ncyc=1000`: 最初の 1000 cycle は steepest descent、それ以降は conjugate gradient

## 4. 入力ファイル `01_Min.in` の解説

[`01_Min.in`](./01_Min.in) のパラメータ一覧:

| パラメータ | 値 | 何を意味するか |
|-----------|----|--------------|
| `imin` | `1` | 1 = エネルギー最小化、0 = MD。今回は最小化なので 1 |
| `ntx` | `1` | 入力ファイルから座標のみ読む (速度はないので) |
| `irest` | `0` | 新規実行 (時間カウンタを 0 から開始。最小化なので意味は薄いが慣例) |
| `maxcyc` | `2000` | 最大 cycle 数 |
| `ncyc` | `1000` | この cycle までは steepest descent、超えたら conjugate gradient |
| `ntpr` | `100` | 100 cycle ごとに `.out` にエネルギーを書く |
| `ntwx` | `0` | 0 = トラジェクトリ (`.nc`) を書かない (最小化なので動きを記録する意味がない) |
| `cut` | `8.0` | nonbonded カットオフ 8 Å (詳細は [GLOSSARY](../../GLOSSARY.md#カットオフ-cutoff)) |

### Fortran namelist 形式って何?

AMBER の入力ファイルは **Fortran namelist** という古い形式です:

```
[タイトル行]
 &cntrl
  変数1 = 値1,
  変数2 = 値2,
  ...
 /
```

- `&cntrl` で始まる
- `/` で終わる
- 変数代入はカンマ区切り
- インデント・改行は自由 (整形は読みやすさのため)
- `!` から行末まではコメント

これは sander/pmemd の入力で全部共通です。

## 5. 実行してみる

```bash
# このリポジトリのルートから
cd workspace
mkdir -p 02_minimization && cd 02_minimization

# 入力ファイルをコピー (もしくは中身を見て自分で書く)
cp ../../solutions/02_minimization/01_Min.in .

# sander を実行
sander -O \
    -i 01_Min.in \
    -o 01_Min.out \
    -p ../01_setup/diala.parm7 \
    -c ../01_setup/diala.rst7 \
    -r 01_Min.ncrst \
    -inf 01_Min.mdinfo
```

通常は数秒〜十数秒で終わります。

### sander のコマンドラインオプション

| フラグ | 役割 | 今回の値 |
|-------|------|---------|
| `-O` | Overwrite — 既存の出力を上書き | (フラグのみ) |
| `-i` | input — 制御パラメータ | `01_Min.in` |
| `-o` | output — テキストログ | `01_Min.out` |
| `-p` | parameter — トポロジー | `../01_setup/diala.parm7` |
| `-c` | coordinate — 入力座標 | `../01_setup/diala.rst7` |
| `-r` | restart — 最終座標の出力先 | `01_Min.ncrst` |
| `-inf` | mdinfo — 進捗情報 | `01_Min.mdinfo` |

`-x` (トラジェクトリ出力) は最小化なので不要 (`ntwx=0` だから書かれない)。

## 6. 出力の見方

```
02_minimization/
├── 01_Min.in        # 入力
├── 01_Min.out       # ★各 cycle のエネルギー履歴 (人が読む)
├── 01_Min.ncrst     # ★最小化後の座標 (次のステップで使う)
└── 01_Min.mdinfo    # 実行中の進捗情報
```

### `01_Min.out` の中身

ヘッダの後に、こんな表が定期的に出てきます:

```
   NSTEP       ENERGY          RMS            GMAX         NAME    NUMBER
      1   1.5234E+04     ...           ...
    100   ...            ...           ...
    200   ...            ...           ...
   ...
   2000  -1.8721E+04     ...           ...
```

各列の意味:

| 列名 | 意味 |
|------|------|
| `NSTEP` | 何 cycle 目か |
| `ENERGY` | 系全体のポテンシャルエネルギー (kcal/mol) |
| `RMS` | 力のルート平均二乗 (RMS gradient)。0 に近いほど平衡 |
| `GMAX` | 最大の力。最も「ひずんでいる」原子の力 |
| `NAME` / `NUMBER` | その最大力を担っている原子の名前と番号 |

### 健全な収束パターン

```
NSTEP    ENERGY
   1    +15234   ← 初期 (大きく正、ひずんでいる)
 100    +800     ← 急速に下がる
 500    -10000
1000    -17000   ← steepest descent 終了
1500    -18500   ← conjugate gradient で精密化
2000    -18721   ← ほぼ収束
```

ポイント:

- ✅ 最初の数十 cycle で **エネルギーが急速に下がる** (steepest 段階)
- ✅ 1000 cycle 過ぎでさらに少し下がる (conjugate gradient 段階)
- ✅ 最後は **負の値で安定** (-1万〜-2万 kcal/mol。水箱のサイズ次第)
- ✅ `RMS` も同時に下がっている (0.1〜1 kcal/mol/Å 程度に)

### `01_Min.out` の最後の方に出る `FINAL RESULTS` セクション

```
                    FINAL RESULTS

   NSTEP       ENERGY          RMS            GMAX         NAME    NUMBER
   2000     -1.8721E+04   8.7654E-02   3.2145E+00     HW1     451

 BOND    =      135.32  ANGLE   =     186.21  DIHED      =      96.45
 VDWAALS =     2837.45  EEL     =  -23012.99  EHBOND     =       0.00
 1-4 VDW =       28.34  1-4 EEL =      -23.10  RESTRAINT  =       0.00
```

これがエネルギー成分の内訳です:

| 成分 | 意味 | 値の目安 |
|------|------|---------|
| `BOND` | 結合の伸縮エネルギー | 数百 kcal/mol (正、ゼロから少しズレ) |
| `ANGLE` | 結合角の歪みエネルギー | 数百 |
| `DIHED` | 二面角のねじれエネルギー | 数十〜数百 |
| `VDWAALS` | van der Waals (Lennard-Jones) | 数千 (正なら反発が強い、負なら引力勝ち) |
| `EEL` | 静電 (Coulomb 含む) | **大きな負** (溶媒和エネルギー) |
| `1-4 VDW`, `1-4 EEL` | 4 結合離れた原子間の特別な相互作用 (ペプチドの重要な成分) | 数十 |
| `RESTRAINT` | 拘束エネルギー (今回はなし) | 0 |

`EEL` が大きく負になるのが普通 (水分子間の双極子相互作用が支配的)。この内訳を見ると「**何が下がってエネルギーが下がったのか**」が分かります。

## 7. 出力の品質チェック

実行後、最低限これだけ見ます:

```bash
# 最終結果を抜き出す
grep -A 8 "FINAL RESULTS" 01_Min.out

# エネルギーの推移
grep -E "^ +[0-9]+ +-?[0-9]" 01_Min.out | head -20
```

健全な兆候:
- 最終 ENERGY が初期から 1〜2 桁減っている
- `VDWAALS` が極端に大きな正の値ではない (= ぶつかりが解消された)
- `RMS` が 1 kcal/mol/Å 以下くらいに収束

## 8. つまずきポイント

| 症状 | 原因と対処 |
|------|-----------|
| `Energy too high at step ...` で死ぬ | tleap の段階で深刻なクラッシュ。`solvateoct` のバッファを 12.0 などに広げてみる |
| エネルギーが下がらない・振動する | `cut` を 10.0 に広げる、もしくは `maxcyc=5000` に増やす |
| `parm7` ファイルが見つからない | sander は `-p` のパスを実行ディレクトリ基準で解釈。`../01_setup/diala.parm7` のパスが合っているか確認 |
| 出力が書けない | `-O` フラグ忘れ (既存ファイルがあるとデフォルトで上書きしない) |

## 9. 用語ミニ辞典 (このステップ初出)

- **ポテンシャルエネルギー曲面 (PES)**: 全原子座標を入力すると 1 つの数値 (エネルギー) を返す関数。最小化はこの曲面を下る作業
- **steepest descent / conjugate gradient**: 最小化アルゴリズム。詳細 [GLOSSARY § D](../../GLOSSARY.md#エネルギー最小化-energy-minimization)
- **RMS gradient**: 力 (= -dE/dr) のルート平均二乗。0 に近いほど平衡点
- **EEL / VDWAALS**: 静電エネルギー / van der Waals エネルギーの内訳
