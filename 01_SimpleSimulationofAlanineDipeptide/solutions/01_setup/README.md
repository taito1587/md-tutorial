# Step 01 — System setup (tleap で系を組み立てる)

> このステップで必要な前知識: なし (初出の用語はここで全部説明します。さらに詳しくは [GLOSSARY](../../../GLOSSARY.md) を参照してください)

## このステップで何をやるか

**アラニンジペプチド** (ACE-ALA-NME) という小さな分子を組み立てて、**水で囲んだ箱**に入れて、後段の MD シミュレーションが始められる状態にします。

具体的なアウトプットは:

- `diala.parm7` — 系のトポロジーと力場パラメータをまとめたファイル
- `diala.rst7` — 全原子の初期座標

この 2 つを次のステップ 02 (最小化) に渡します。

---

## 1. 前提知識 — 何のためにこの作業が必要なのか

### MD シミュレーションには 3 つの情報が必要

第 1 章で書いた MD の計算ループを思い出してください:

```
位置 r → 力場で力 F を計算 → 速度を更新 → 位置を更新
```

これを動かすには最低限以下の 3 つが要ります:

1. **どの原子があってどう繋がっているか** (= **トポロジー**)
2. **力場パラメータ** (= 各結合・各原子ペアでどれくらいの力が働くかの数値)
3. **初期座標** (= 計算をスタートする瞬間の各原子の位置)

このステップ 01 は、**この 3 つを全部用意するための準備工程**です。

### tleap の役割

[`tleap`](../../../GLOSSARY.md#tleap) は AMBER の「**分子ビルダー**」です。

- 力場ライブラリ (`leaprc.protein.ff19SB` 等) を読み込む
- 残基名のリストから分子を組み立てる
- 周りに水分子を並べて溶媒和する
- 出来上がった系のトポロジー + 座標をファイルに保存する

をスクリプト 1 本でやってくれます。

## 2. 系の正体 — アラニンジペプチドって何?

### 構造

```
       ACE             ALA              NME
   ┌──────────┐    ┌──────────┐    ┌──────────┐
   CH3-C(=O)─── NH-CHα(CH3)-C(=O)─── NH-CH3
       │           │              │
       (C 末端     (アミノ酸の中央)  (N 末端
       キャップ)                    キャップ)
```

合計 22 原子。タンパク質の途中に出てくるアラニン残基をそのまま切り取ったような分子です。

### なぜ「ジペプチド」なのにアミノ酸 1 個なのか

歴史的事情なのですが、**ペプチド結合 (CO-NH 結合) が 2 本**含まれているので「ジペプチド」と呼ばれます。アミノ酸の数ではなくペプチド結合の数で数えるのですね。

### なぜキャップ (ACE / NME) を付けるのか

タンパク質の中の残基は両側にも他の残基が繋がっています。その化学環境を再現するため、両端を**小さな「ふた」で塞ぎ電気的に中和**しておきます。

- もし生のアラニンを切り出したら、N 末端は `+H3N-`、C 末端は `-COO-` で **両端に電荷**を持ってしまう
- これだとタンパク質中央の中性的な残基とは振る舞いが大きく変わる
- そこで小さなキャップ (ACE と NME) を付けて電荷をなくす

### なぜこの分子で勉強するのか

| 理由 | 詳細 |
|------|------|
| **計算が速い** | 全原子 22 個 + 水。10 ns の MD が CPU でも 1 日以内で終わる |
| **二面角 φ/ψ が 1 セット** | タンパク質の二次構造を決める重要な角度を最も簡単な形で観察できる |
| **歴史的標準** | MD 開発の検証に必ず使われる「ベンチマーク分子」 |

## 3. 力場と水モデルの選択

### ff19SB (タンパク質用)

[力場](../../../GLOSSARY.md#力場-force-field) はざっくり言うと「**この原子間にはこういう力が働く**」を全部まとめた数値テーブル。タンパク質用の力場は何種類もありますが、**ff19SB** (2020 年公開) が現在の AMBER 公式推奨です。

- backbone (主鎖) の挙動が実験 NMR データに合うようにフィッティングされている
- **OPC 水と組み合わせて使う前提**で開発されている

### OPC 水モデル

水分子を MD でどう表現するか、にもいくつか流派があります:

| 水モデル | 力 (相互作用) サイト | 特徴 |
|----------|------------------|------|
| TIP3P | 3 (O, H, H) | 古典的。軽いが密度・誘電率がやや低め |
| TIP4P/Ew | 4 (+ ダミー) | 中間 |
| **OPC** | 4 (+ 仮想 M サイト) | **誘電率・自己拡散が実験に近い**。今回使う |

OPC は H, H, O の 3 つの実原子に加えて、電荷を持つ「**仮想粒子 M**」を 1 つ追加した「4 点モデル」。これにより電荷分布が実際の水分子に近くなります。

> ⚠️ 「**ff19SB は必ず OPC と組み合わせる**」ことが AMBER 公式推奨です。ff19SB を TIP3P で使うと結果がおかしくなることが知られています。

## 4. tleap スクリプトの中身を行ごとに

[`build.leap.in`](./build.leap.in) の各行が何をしているか:

```text
source leaprc.protein.ff19SB
```
タンパク質用の力場 (ff19SB) を読み込む。これで標準アミノ酸 20 種類 + キャップの定義が tleap に教え込まれます。

```text
source leaprc.water.opc
```
水モデル OPC を読み込む。水分子の構造定義 + プリ平衡化された箱型クラスタ `OPCBOX` が利用可能になります。

```text
diala = sequence { ACE ALA NME }
```
変数 `diala` に「ACE-ALA-NME という配列で組み立てた分子」を代入。tleap が必要な原子を配置・結合します。

```text
solvateoct diala OPCBOX 10.0
```
**`solvateoct`**: solvate in a truncated **oct**ahedron (切頭八面体) 形の箱で溶媒和する。

- `diala`: 溶質の分子
- `OPCBOX`: 使う水のテンプレート (OPC 水のプリ平衡済みクラスタ)
- `10.0`: 溶質の最も外側の原子から箱の縁までの最低距離 (バッファ) を **10 Å** にする

切頭八面体は立方体より体積が小さくて済むので、**同じバッファでも水分子の数が 25% くらい少ない** = 計算が速い。よく使われます。

```text
saveamberparm diala diala.parm7 diala.rst7
```
組み立てた系を 2 つのファイルに書き出す:
- `diala.parm7` — トポロジー + 力場パラメータ (テキスト)
- `diala.rst7` — 初期座標 (ASCII NetCDF)

> 💡 **公式チュートリアルではファイル名を `parm7` / `rst7` (拡張子なし) としています**が、本リポジトリではより一般的な `diala.parm7` / `diala.rst7` という命名にしました。中身は同じです。後続のコマンドはすべてこの命名で揃えてあります。

```text
quit
```
tleap を終了。

## 6. 出力の見方

実行すると以下のファイルができます:

```
01_setup/
├── build.leap.in    # 入力 (自分でコピー)
├── diala.parm7      # ★トポロジー + 力場 (次以降のステップで -p で渡す)
├── diala.rst7       # ★初期座標 (次のステップで -c で渡す)
└── leap.log         # tleap の実行ログ全文
```

### tleap の標準出力で見るべきところ

実行末尾にこんな表示が出ます:

```
> solvateoct diala OPCBOX 10.0
Scaling up box by a factor of 1.000000 to meet diagonal cut criterion
  Solute vdw bounding box:              5.700  4.220  4.940
  Total bounding box for atom centers:  31.700 30.220 30.940
  Total vdw box size:                   33.460 31.991 32.704
  Volume:  35021.42 A^3
  Total mass 18156.31 amu, Density 0.86 g/cc
  Added 998 residues.
```

| 項目 | 何を意味するか |
|------|--------------|
| Solute vdw bounding box | 溶質の van der Waals サイズ。アラニンジペプチドが約 6 × 4 × 5 Å |
| Total bounding box | バッファ込みの箱サイズ |
| Volume | 箱の体積 (Å³) |
| Density | この時点の密度 (g/cc)。0.86 程度ならまだ MD 平衡前なので OK |
| **Added X residues** | 追加された水分子の数。アラニンジペプチド (3 残基) と合わせて全 1001 残基くらい |

### `leap.log` を見る

```bash
less leap.log
```

`Error` で grep すると問題ないか確認できます:

```bash
grep -i error leap.log
```

何も引っかからなければ正常です。`Warning` はだいたい無視 OK ですが、メッセージ内容は読んでおくとよいです。

## 7. 何が起きているか — 中身をもう少し覗く

`diala.parm7` をテキストエディタで開くと、ヘッダ部分にこんな構造が並んでいます:

```
%FLAG TITLE
%FLAG POINTERS
   3001    NATOM     ← 全原子数 ~3000
   3       NRES      ← まずジペプチドの 3 残基だけ表示...
%FLAG ATOM_NAME
%FLAG CHARGE         ← 各原子の電荷 (×18.2223 倍されて格納)
%FLAG MASS           ← 質量
%FLAG BONDS_INC_HYDROGEN  ← 結合のリスト
...
```

このファイルを **MD 中ずっと参照する** ことで、力場が機能します。

`diala.rst7` には単に初期座標が書かれているだけです:

```
default name
  3001
  10.000  10.123  9.876   ← atom 1 の x, y, z (Å)
  10.523  10.001  9.534   ← atom 2 の x, y, z
  ...
```

各原子について `(x, y, z)` が並ぶシンプルな形式。これをスタートに MD が動き出します。

## 8. つまずきポイント

| 症状 | 原因と対処 |
|------|-----------|
| `Could not open file leaprc.protein.ff19SB` | AmberTools の環境変数が読めていない。`nix develop` で入ったシェル内で実行しているか確認 |
| `Could not find type for OW` のような warning | OPC 水の force field を読む前に `solvateoct` を呼んでいる可能性。`source leaprc.water.opc` の順序を確認 |
| `Failed to generate parameters` | 残基名のスペルミス (ACE と AC、NME と NMA など)。`sequence { }` の中を確認 |
| 出力ファイル名が違う | `saveamberparm` の引数: 第 1 引数 = 変数名、第 2 引数 = parm7 のファイル名、第 3 引数 = rst7 のファイル名 |

## 9. 用語ミニ辞典 (このステップで初出のもの)

- **トポロジー**: 各原子の質量・電荷・結合関係などをまとめたメタ情報。MD 中ずっと参照される。詳細 [GLOSSARY](../../../GLOSSARY.md#parm7--prmtop)
- **力場 (force field)**: 原子間の相互作用エネルギーの式 + そのパラメータ群。詳細 [GLOSSARY § B](../../../GLOSSARY.md#b-力場とエネルギー)
- **OPC 水**: 4 点モデルの水分子。詳細 [GLOSSARY](../../../GLOSSARY.md#opc-水)
- **溶媒和 (solvate)**: 溶質を水分子で取り囲むこと
- **切頭八面体 (truncated octahedron)**: 立方体の 8 隅を斜めに削った周期境界箱。少ない水で済む
