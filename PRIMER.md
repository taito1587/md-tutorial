## 1. MD シミュレーションとは

### ひとことで言うと

> **コンピュータの中に分子の世界を作って、Newton の運動方程式を解いて、どう動くかを見るシミュレーション**

です。

### もうちょい丁寧に

実験では「タンパク質が水中でどう揺れているか」を 1 原子レベルで見るのは難しい (X 線結晶構造解析は静止画、NMR は時間平均)。一方コンピュータの中なら、

1. 全部の原子の位置を知っている
2. 全部の原子に働く力も計算できる
3. F = ma で次の瞬間の位置が決まる

ので、**原理的には全原子の動きを 1 fs (= 10⁻¹⁵ 秒) 刻みで追跡できる**。実験では絶対に見えない時間スケールが、計算なら見える ── これが MD の魅力です。

### 適用範囲と限界

| MD で見えるもの | MD で見えないもの |
|----------------|------------------|
| 分子のゆらぎ・形状変化 | 化学反応 (結合の生成・切断) |
| タンパク質の柔軟性 | 量子効果 (トンネル効果、電子状態) |
| 結合エネルギー (相対値) | 紫外〜可視領域の励起状態 |
| 拡散・粘性 | 1 μs を超える長時間スケール (普通の MD では届きにくい) |

## 2. なぜ MD をやるのか

### 例 1: 創薬

候補化合物がタンパク質の活性ポケットに**どれくらいぴったりハマるか**を MD で評価する。実験合成の前にスクリーニング。

### 例 2: 構造生物学の補強

X 線で得た静止画を出発点に、**どう揺れているか・どう開閉するか**を見る。

### 例 3: 物理化学

水の自己拡散係数や粘性、脂質二重膜の厚さといった**マクロ物性**を分子レベルから理解する。

このチュートリアル 0 のアラニンジペプチドはこれら全部の入り口。**最小のペプチドモデル**で、まず MD のパイプラインを一通り動かせるようになることが目標です。

## 3. MD の中身 — 何を計算しているのか

### 大まかな流れ

```
┌─────────────────────────────────────────┐
│ 1. すべての原子の位置 r と速度 v を持つ   │
└────────────────┬────────────────────────┘
                 ▼
┌─────────────────────────────────────────┐
│ 2. 力場 (force field) で各原子の力 F を  │
│    位置 r から計算する                   │
└────────────────┬────────────────────────┘
                 ▼
┌─────────────────────────────────────────┐
│ 3. F = ma → a = F/m                      │
│    速度を Δt = 2 fs だけ進める          │
│    座標を Δt だけ進める                  │
└────────────────┬────────────────────────┘
                 ▼
              (戻る)
```

これを **数百万回〜数千万回** 繰り返します。10 ns シミュレーションなら 5,000,000 回。

### 力場 (force field) — 力の計算式

各原子に働く力 F = -∇E は、エネルギーを座標で微分すれば出てきます。エネルギーは:

```
E = (結合の伸縮)
  + (結合角のひずみ)
  + (二面角のねじれ)
  + (van der Waals 反発・引力)
  + (静電相互作用)
```

の和。**結合は単純なバネ**、**ねじれは cos の和**、**遠距離はクーロン則と Lennard-Jones**、と思いっきり古典力学で近似しています。**化学反応は扱えない**のはここが理由 (結合の切れ方を表現できる関数になっていない)。

詳細は [GLOSSARY § B](GLOSSARY.md#b-力場とエネルギー) を参照。

### なぜ 2 fs 刻みなのか

水素を含む結合 (O-H, C-H など) の振動周期が約 10 fs。Nyquist の定理的に、この振動を捉えるには 1 fs 以下の刻みが必要。**でも、SHAKE (アルゴリズム) で結合長を固定すれば、もはや振動を計算しなくてよくなり、2 fs まで伸ばせる**。これが MD のスタンダード設定。

## 4. このチュートリアルで作る系 — アラニンジペプチド

### 系の正体

アミノ酸 1 個 (アラニン = ALA) の両端を ACE / NME という小さなキャップで塞いだ分子です。

```
   ACE          ALA           NME
  ┌─────┐    ┌──────┐    ┌─────┐
  CH3-CO──── NH-CHα-CO ──── NH-CH3
                │
                CH3 (側鎖)
```

合計 22 原子のとても小さな分子。これを水で溶媒和 (solvate) して、約 1500〜3000 原子の系を作ります (大半は水)。

### なぜアラニンジペプチドなのか

- **小さい** → 計算が速い、教材に最適
- **両端キャップ** → タンパク質中のアミノ酸残基と化学環境が同じになる
- **二面角 φ/ψ が 1 セット** → Ramachandran プロット解析の基本例

これで MD の流儀を覚えたら、次は溶媒中のタンパク質 (例: lysozyme) や DNA、薬剤・タンパク質複合体へと進みます。

## 5. このチュートリアル全体の見通し

5 ステップで進みます。各ステップで「**何を入力に、何を出力するか**」だけ最初に押さえると迷子になりません。

### Step 01 — System setup (`solutions/01_setup/`)

**やること**: アラニンジペプチドを組み立てて、水で溶媒和する。
**ツール**: `tleap`
**入力**: なし (スクリプト `build.leap.in` 内ですべて指定)
**出力**: `diala.parm7` (トポロジー), `diala.rst7` (初期座標)

### Step 02 — Energy minimization (`solutions/02_minimization/`)

**やること**: 初期構造のひずみをエネルギー的に解消する。動かすわけではなく、座標を最適化するだけ。
**ツール**: `sander`
**入力**: `diala.parm7`, `diala.rst7`
**出力**: `01_Min.ncrst` (最小化後の座標), `01_Min.out` (エネルギーログ)

### Step 03 — Heating (`solutions/03_heating/`)

**やること**: 0 K から 300 K まで 20 ps かけてゆっくり加熱する MD。
**ツール**: `sander`
**入力**: `diala.parm7`, `01_Min.ncrst`
**出力**: `02_Heat.ncrst`, `02_Heat.nc` (トラジェクトリ), `02_Heat.out`

### Step 04 — Production MD (`solutions/04_production/`)

**やること**: 300 K, 1 atm の条件下で 10 ns の本番 MD。
**ツール**: `pmemd` (or `sander`)
**入力**: `diala.parm7`, `02_Heat.ncrst`
**出力**: `03_Prod.ncrst`, `03_Prod.nc`, `03_Prod.out`

### Step 05 — Analysis (`solutions/05_analysis/`)

**やること**: 温度・密度・RMSD をプロットしてシミュレーションが妥当か確認。
**ツール**: `cpptraj`, `process_mdout.perl`, `gnuplot` (公式は xmgrace。本リポジトリは Apple Silicon 互換性で gnuplot を使用)
**入力**: ステップ 03/04 の `.nc` と `.out`
**出力**: `summary.TEMP`, `summary.DENSITY`, `02_03.rms` ほか

### 図でまとめると

```
[Step 01]              [Step 02]             [Step 03]              [Step 04]               [Step 05]
                                                                                            
build.leap.in     →   01_Min.in     →     02_Heat.in       →    03_Prod.in       →     plot.sh
                                                                                            rmsd.cpptraj
   tleap                sander                sander                  pmemd                  cpptraj
                                                                                            process_mdout.perl
                                                                                            gnuplot
   ↓                     ↓                     ↓                      ↓                      ↓
diala.parm7         01_Min.ncrst          02_Heat.ncrst           03_Prod.ncrst          summary.TEMP
diala.rst7                                02_Heat.nc              03_Prod.nc             summary.DENSITY
                                                                                          02_03.rms
                                                                                          (グラフ)
```

## 6. なぜこの 5 ステップがこの順番なのか

「最小化 → 加熱 → 本番」の順は **必ずこの順** です。なぜか:

1. **`tleap` の出した初期構造はそのままでは MD にかけられない**。原子同士がぶつかりかけている部分があり、いきなり MD すると爆発する。→ **最小化**でひずみを取る。
2. **最小化の出力は「動いていない構造」**。いきなり 300 K の運動エネルギーを与えるとホットスポットができて壊れる。→ **加熱**でゆっくり温度を上げる。
3. **加熱は密度が変わる**ので NVT (体積一定) でやる。本番では現実条件 (1 atm) で見たいので **NPT (圧力一定)** に切り替え。
4. **本番 MD のトラジェクトリ**を解析して、温度や RMSD がもっともらしいか確認 (= 結果が信頼できるか判断)。

この preparation → equilibration → production という流れは **MD の鉄則**で、どんなプロジェクトでも同じ。今後どんなタンパク質を扱うようになっても踏襲します。
